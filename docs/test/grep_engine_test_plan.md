# Grep エンジン テスト計画書

Grepエンジンの各機能が正しく動作することを確認するためのテスト項目です。
テストは `pytest` を使用して自動化します。

## 1. ユニットテスト項目 (`tests/test_grep_engine.py`)

### A. 基本スキャン機能
- [ ] 指定されたテキストがファイル内に1つある場合に正しく検出できるか
- [ ] 同一ファイル内に複数のヒットがある場合に、すべての行が検出されるか
- [ ] 検索テキストが含まれていない場合にヒット数が0になるか

### B. 正規表現モード
- [ ] `regex_mode=True` のときに正規表現パターンで検索できるか (例: `\d{3}-\d{4}`)
- [ ] 不正な正規表現（閉じられていない括弧など）が指定された場合でもクラッシュせず、通常の検索またはエラーハンドリングが行われるか

### C. 多言語/文字コード対応（外部ライブラリ不使用の挙動確認）
- [ ] `utf-8` でエンコードされた日本語テキストを検出できるか
- [ ] `cp932` (Shift-JIS) でエンコードされた日本語テキストを検出できるか
- [ ] `euc_jp` でエンコードされた日本語テキストを検出できるか
- [ ] どのエンコーディングでもデコードできないファイル（純粋なバイナリ）でエラーが発生せず、無視されるか

### D. 並列処理・コールバック
- [ ] `on_progress` が正しい合計数と現在のスキャン数を通知するか
- [ ] `on_result` が見つかった順番に正しく呼び出されるか
- [ ] `on_complete` が最終的なヒット数を正しく返すか
- [ ] 大量のファイル（100個以上）を検索しても、デッドロックやクラッシュを起こさないか

### E. 中断機能 (Cancellation)
- [ ] `stop()` メソッドを呼び出した際、それ以降のファイル走査が速やかに停止するか

### F. Officeファイル検索 (Word/Excel)
- [ ] `.docx` ファイル内部のテキストを検知し、ヒット行として取得できるか
- [ ] `.xlsx` ファイル内の共有文字列 (sharedStrings.xml) を検知し、ヒット行として取得できるか
- [ ] 本文が「[Office]」プレフィックス付きで返ってくるか
- [ ] 不正な形式のZIP/Officeファイルが含まれていてもクラッシュせずスキップされるか

## 2. 統合テスト項目
- [ ] `cli_test.py` を通じて、実際のプロジェクト構造を対象に正確な検索が行えるか
